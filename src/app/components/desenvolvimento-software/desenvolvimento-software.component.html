V6: Empréstimo consignado

Simulador de empréstimo consignado - Sexta versão: Programação Orientada a Objetos

Java POO | EMPRÉSTIMO

Este sistema simula um processo de autenticação e contratação de empréstimos, utilizando Programação Orientada a Objetos (POO) em Java. O código está organizado em pacotes, classes, métodos, atributos e encapsulamento, seguindo a estrutura modular para cada parte do processo. Vou dividir a análise do programa de acordo com os pacotes e classes fornecidos.

Estrutura do Programa
O sistema é composto por três pacotes principais:
application: Onde a classe principal Simulador executa o fluxo do programa, interagindo com o usuário.
financas: Contém a classe Contrato, que lida com as operações financeiras do sistema, como cálculo de valores de empréstimos e comissões.
usuarios: Contém a classe Autenticacao, responsável pela autenticação do usuário.
Pacote application - Classe Simulador
- Objetivo: Controla o fluxo principal da aplicação, coordenando o cadastro de usuários, login e escolha de operações bancárias e financeiras.
- Atributos: Usa instâncias das classes Autenticacao e Contrato.
- Métodos:
- main(String[] args): Método principal onde ocorre a interação com o usuário. Primeiramente, chama o processo de cadastro e login do sistema (via classe Autenticacao), seguido pela escolha de bancos e operações financeiras (via classe Contrato).
- Cadastro Inicial e Login: Ao executar o programa, o método realiza a autenticação do usuário e solicita as credenciais.
- Seleção de Banco: Após o login, o usuário escolhe o banco desejado, o qual é responsável por ajustar a comissão e o limite de idade para o processo.
Seleção de Operação: O usuário pode escolher entre realizar um novo empréstimo ou refinanciamento de dívida.
Pacote financas> - Classe Contrato>
- Objetivo: Gerenciar as operações financeiras, como a simulação de empréstimos, refinanciamentos e cálculos de comissão de corretor.
- Atributos:
- Informações do cliente: Atributos como nomeCliente, cpfCliente, idadeCliente, sexoCliente, etc.
- Operação financeira: Atributos relacionados ao valor do empréstimo, parcela, comissão e prazos.
- Variáveis de cálculo: Como porcentagemComissao, fator (para cálculos de valores de empréstimos), e valorLiquido.
Métodos:
- bancoBradesco(), bancoItau()>, etc.: Métodos para selecionar um banco específico, cada um com sua comissão e limite de idade para aprovação de crédito.
- DadosDoCliente(): Coleta os dados do cliente necessários para simular um empréstimo.
- ContratoNovo() e ContratoDeRefinanciamento(): Métodos que simulam a criação de um novo empréstimo ou refinanciamento de dívida, onde o usuário escolhe como deseja realizar a operação (por parcela ou valor do contrato).
- ComissaoCorretor(): Método para calcular a comissão do corretor com base no banco selecionado e no valor do empréstimo.
- Resultado(): Exibe os detalhes do contrato simulado, incluindo o valor da parcela, comissão do corretor, entre outros.
Pacote usuarios> - Classe Autenticacao
- Objetivo: Gerenciar o processo de autenticação do usuário, incluindo cadastro e login.
- Atributos:
- nome, usuario, senha: Informações de cadastro do usuário.
- repetirUsuario, repetirSenha: Variáveis temporárias para verificar se os dados de login fornecidos estão corretos.
- erros: Contabiliza os erros de login.
Métodos:
- cadastroInicial(): Realiza o cadastro do usuário (nome, usuário, senha), permitindo que o sistema armazene essas informações para autenticação futura.
acessarSistema()>: Verifica as credenciais fornecidas pelo usuário. Caso o usuário erre a senha mais de 2 vezes, o sistema solicita um novo cadastro.

Encapsulamento e Atributos
O código segue os princípios de encapsulamento ao fornecer métodos getters e setters para os atributos privados de cada classe. Isso permite que os dados dos objetos sejam acessados e manipulados de forma controlada e segura. Por exemplo, os atributos de cliente na classe Contrato (nomeCliente, cpfCliente>, etc.) são acessados por meio dos métodos get> e set>, mantendo a integridade dos dados.

Interação com o Usuário
O programa faz uso intenso de Scanner> para capturar a entrada do usuário, seja para o cadastro, login ou dados relacionados aos contratos financeiros. O fluxo de interação é simples e intuitivo, com validações de entrada, como o número máximo de tentativas de login.

Em resumo, o programa simula um processo de cadastro, login e simulação de empréstimos/refinanciamentos de forma bem estruturada utilizando Programação Orientada a Objetos. Ele segue a modularização, com diferentes responsabilidades divididas em pacotes e classes, e emprega o uso de encapsulamento para garantir segurança no acesso aos dados. O fluxo principal da aplicação começa com o cadastro e autenticação do usuário, segue com a escolha de um banco e operação financeira, e termina com a simulação de um contrato, incluindo o cálculo de parcelas, valores liberados e comissão do corretor.

V5: Empréstimo consignado

Simulador de empréstimo consignado - Quinta versão

C# | EMPRÉSTIMO

O programa implementa um sistema de cadastramento e simulação de empréstimos consignados e refinanciamentos, com controle de usuário e senha, além de verificações de requisitos mínimos para cada operação.
1.Cadastro e Autenticação:
- O programa começa com o cadastro de um usuário e senha.
- O usuário precisa fazer login para acessar o sistema. Se o login for bem-sucedido, o programa prossegue.
2.Cadastro de Operações:
- O usuário é solicitado a inserir a quantidade de operações que deseja cadastrar.
- Para cada operação, o programa solicita informações sobre o banco, data (dia e mês) e a escolha do órgão (INSS, SIAPE ou Exército).
- A comissão para cada órgão pode ser personalizada pelo usuário.
3.Opções de Operação:
- O usuário escolhe entre dois tipos de operação: Nova operação ou Refinanciamento.
- Nova operação:
- O usuário fornece os dados do cliente (nome, CPF, valor bruto, valor da parcela e prazo).
- Dependendo do valor bruto informado e do órgão escolhido, o sistema verifica se o valor está abaixo do mínimo permitido. Caso esteja, a operação é negada. Caso contrário, são apresentados os detalhes da operação e a comissão a ser paga.
- Refinanciamento:
- O processo é similar à nova operação, mas inclui o valor do saldo devedor.
- O valor liberado é calculado subtraindo o saldo devedor do valor bruto.
- Novamente, o programa verifica se o valor liberado está abaixo do mínimo exigido por órgão, negando a operação, caso necessário.
4.Verificação de Requisitos:
- Para cada tipo de operação (nova ou refinanciamento), o programa verifica se o valor informado (bruto ou liberado) atende ao mínimo permitido para o órgão selecionado:
- INSS: Mínimo de R$ 500 para nova operação e R$ 300 para refinanciamento.
- SIAPE: Mínimo de R$ 850 para nova operação e R$ 400 para refinanciamento.
- Exército: Mínimo de R$ 1.000 para nova operação e R$ 500 para refinanciamento. 5.Repetição de Cadastro:
- O programa usa um laço de repetição (for) para permitir que o usuário cadastre várias operações de uma vez, conforme a quantidade definida no início.
Resultado:
- Caso os requisitos sejam atendidos, o programa exibe os detalhes da operação, incluindo o valor da comissão, o valor bruto, as parcelas e o prazo.
Encerramento:
- Ao final do processo, o programa exibe uma mensagem de encerramento e aguarda uma ação do usuário para finalizar.

Essa versão do programa se destaca pela introdução do controle de autenticação, permitindo que apenas usuários cadastrados possam acessar as funcionalidades. Além disso, o uso da estrutura de repetição permite que múltiplas operações sejam cadastradas de forma eficiente.

V4: Empréstimo consignado

Simulador de empréstimo consignado - Quarta versão

C# | EMPRÉSTIMO

Esta é a quarta versão do simulador de empréstimo consignado, agora implementada em C#. O programa segue a mesma lógica das versões anteriores, mas foi adaptado para a linguagem C#. Ele permite ao usuário selecionar um órgão (INSS, SIAPE ou Exército), definir o tipo de operação (novo empréstimo ou refinanciamento) e inserir dados como nome, CPF, valor bruto, saldo, parcela e prazo. O sistema então calcula a comissão com base na porcentagem associada ao órgão e operação escolhidos, e exibe os resultados ao usuário.

Principais Características e Funcionalidade:

1.Entrada de Dados: O programa começa solicitando ao usuário que escolha um órgão para o cadastramento (INSS, SIAPE ou Exército). Em seguida, o usuário deve selecionar o tipo de operação (novo ou refinanciamento) e fornecer dados como nome, CPF, valor bruto do empréstimo, saldo, valor da parcela e o prazo.

2.Estrutura de Controle (Switch e If/Else):
- Switch: O switch é usado para determinar a porcentagem de comissão com base na escolha do órgão (INSS, SIAPE ou Exército).
- If/Else: A estrutura if/else é utilizada para diferenciar o processo de cadastramento entre um novo empréstimo ou um refinanciamento. Para cada tipo de operação, o programa coleta as informações necessárias e calcula o valor da comissão correspondente.
3.Cálculo da Comissão:
- Novo Empréstimo: Para um novo empréstimo, o programa calcula a comissão com base no valor bruto e na porcentagem do órgão. A comissão é apresentada ao usuário com duas casas decimais.
- Refinanciamento: Para refinanciamentos, além dos dados de valor bruto e parcela, o programa solicita o saldo restante. O valor liberado (valor bruto - saldo) é calculado, e a comissão é ajustada com um desconto de 3%, refletindo uma porcentagem menor para refinanciamentos. O valor da comissão é calculado e exibido com duas casas decimais.
4.Saída de Dados: O programa exibe todas as informações inseridas pelo usuário, incluindo nome do cliente, CPF, valor bruto, valor do saldo (para refinanciamentos), valor liberado, valor da parcela, prazo e a comissão calculada. Tudo é formatado para exibição com duas casas decimais, utilizando a cultura "InvariantCulture" para garantir a consistência na formatação de números.

Comparação com as Versões Anteriores (C++, Java):

- Linguagem C#: A versão em C# segue a mesma lógica básica das versões em C++ e Java, mas com adaptações para a sintaxe e características da linguagem C#. A principal diferença é o uso da biblioteca System para entrada e saída de dados, e a formatação numérica específica com CultureInfo.InvariantCulture.
- Entrada e Saída: A entrada de dados é feita por meio de Console.ReadLine() e a saída é realizada com Console.WriteLine(), características típicas de C#. A formatação de números, especialmente para exibir o valor da comissão com duas casas decimais, é feita com o método ToString("F2", CultureInfo.InvariantCulture).
- Controle de Fluxo: O programa mantém a estrutura de controle de fluxo semelhante às versões anteriores, com um switch para selecionar o órgão e if/else para diferenciar as operações de novo empréstimo e refinanciamento.
- Comissão e Cálculos: O cálculo da comissão continua o mesmo, com a comissão sendo calculada de acordo com a porcentagem do órgão, com um ajuste de 3% para refinanciamentos. O uso de variáveis como double para armazenar valores numéricos permite cálculos precisos.

Conclusão: Esta versão em C# do simulador de empréstimo consignado mantém a funcionalidade central das versões anteriores, com melhorias na formatação e a utilização de funcionalidades específicas da linguagem C#. Ela é simples e eficaz para prática dos conceitos básicos da programação, como estruturas de controle (switch, if/else), manipulação de entradas e saídas de dados e formatação de números.

Simulador de empréstimo consignado - Terceira versão

Java | EMPRÉSTIMO

Esta é a terceira versão do simulador de empréstimo consignado, agora implementada em Java. O funcionamento do programa é muito similar às versões anteriores, mas adaptado para a sintaxe e estrutura do Java. O usuário seleciona o órgão (INSS, SIAPE ou Exército) e o tipo de operação (novo empréstimo ou refinanciamento), insere informações como nome, CPF, valor bruto, saldo, parcela e prazo. O sistema então calcula e exibe o valor da comissão com base na porcentagem definida para o órgão e operação.

Principais Características e Funcionamento:

O código solicita ao usuário a escolha do órgão (INSS, SIAPE ou Exército) e, em seguida, o tipo de operação (novo ou refinanciamento). A seguir, o programa coleta os dados necessários como nome, CPF, valor bruto, saldo, parcela e prazo.

2.Estrutura de Seleção (Switch/Case): Assim como na versão em C++, o código usa o switch para determinar o órgão selecionado, atribuindo a porcentagem da comissão correspondente a cada opção (INSS, SIAPE ou Exército).

3.Operações Diferenciadas:
- Novo Empréstimo: Para operações novas, o programa solicita informações sobre o valor bruto, parcela e prazo e, em seguida, calcula o valor da comissão com base na porcentagem do órgão escolhido.

4.Saída de Dados: O programa exibe os resultados da proposta, incluindo o nome do cliente, CPF, valor bruto, parcela, prazo e o valor da comissão, tanto para empréstimos novos quanto refinanciamentos.

Comparação com as versões anteriores (C++):

- Linguagem: Esta versão foi reescrita em Java, o que significa que a estrutura e sintaxe foram adaptadas para o estilo de programação dessa linguagem. A lógica geral do programa é bastante similar às versões anteriores, com pequenas diferenças na forma como os dados são manipulados e impressos.
- Entrada e Saída: Na versão em C++, a entrada de dados era realizada via cin e a saída via cout. Já em Java, a entrada é feita com Scanner, e a saída utiliza System.out.println, que é o método padrão para imprimir no console em Java.
- Controle de Fluxo: Assim como na versão em C++, esta versão também usa switch para selecionar o órgão e if/else para determinar o tipo de operação. A lógica do controle de fluxo é praticamente a mesma, mas a sintaxe do switch e if/else em Java é ligeiramente diferente.
- Cálculo da Comissão: O cálculo da comissão em ambas as versões é o mesmo. Para refinanciamento, a comissão sofre um desconto de 3%, e o cálculo é ajustado para refletir isso.
- Organização e Legibilidade: Embora a lógica de ambas as versões seja muito parecida, a versão em Java possui uma estrutura mais organizada em termos de uso de classes, pacotes e importações (como a importação da classe Scanner). A linguagem Java também exige mais atenção com os tipos de dados e declarações de variáveis (por exemplo, as variáveis String e int precisam ser explicitamente declaradas).

Conclusão: Esta versão em Java mantém o funcionamento do simulador de empréstimo consignado similar ao que foi feito nas versões anteriores, com o uso de switch, if/else e cálculos de comissão. Porém, ela é mais adequada ao estilo de programação em Java, com um controle de fluxo mais estruturado e entradas/saídas específicas para a linguagem.

2.Código mais compacto: O programa foi reduzido em termos de número de linhas, eliminando redundâncias, mas mantendo a mesma funcionalidade.

- Redução de código: A versão 2 tem menos linhas e é mais compacta. Ela mantém a lógica de operação, mas com menos repetição de código, resultando em um código mais conciso e fácil de entender.
- Simplificação do cálculo da comissão: Na versão anterior, o cálculo da comissão era um pouco mais direto, mas a versão 2 ajusta o valor da comissão para refinanciamentos com um desconto de 3%, o que reflete um novo ajuste lógico e torna o código mais adaptável.
- Interface e interação: Ambas as versões oferecem a mesma interação com o usuário, mas a versão 2 mantém as funcionalidades, com um código mais simplificado e de mais fácil manutenção.

Essa versão é uma melhoria significativa em relação à anterior, destacando-se pela simplificação do código e pela eficiência na estruturação, tornando-o mais legível e pronto para evoluções futuras.
